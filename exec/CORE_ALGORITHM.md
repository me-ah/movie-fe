# [발표자료] 핵심 알고리즘 및 기술적 차별점 설명서

MEAH 프로젝트는 단순한 영화 조회를 넘어, 데이터 기반의 정교한 **개인화 큐레이션 엔진**을 구축하는 데 집중했습니다. 발표 시 다음 3가지 핵심 포인트를 강조해 주세요.

---

## 1. 넷플릭스 스타일 홈 큐레이션 (Smart Category Engine)

### 💡 기획 의도: "유저는 스스로 검색하지 않는다."
유저가 지루하지 않게 탐색할 수 있도록, DB의 실제 데이터로부터 **471개의 세부 장르 조합(Mix-Genre)**을 자동으로 분석하여 생성했습니다.

### ⚙️ 기술적 로직 (Averaging Personalization)
- **장르 조합 분석:** '액션'과 '코미디'가 동시에 포함된 영화들을 추출하여 하나의 전용 레일로 묶습니다.
- **실시간 평균 점수 산정:** 유저가 '액션' 점수가 100점, '코미디' 점수가 50점이라면, '액션 코미디' 레일의 점수는 **평균인 75점**으로 실시간 계산됩니다.
- **다이내믹 랭킹:** 매 접속 시마다 유저의 시청 패턴이 반영된 19개 장르 점수를 분석하여, 471개 후보 중 가장 점수가 높은 **상위 27개 맞춤형 레일**을 동적으로 배치합니다.

---

## 2. 지능형 쇼츠 믹스 추천 (Shorts Mix Engine)

### 💡 기획 의도: "취향 저격과 새로운 발견의 균형"
자신의 취향만 계속 보면 금방 질립니다(필터 버블). 이를 방지하기 위해 **6:2:2 큐레이션 믹스** 전략을 도입했습니다.

### ⚙️ 기술적 로직 (6:2:2 Hybrid Strategy)
매 20개 영상 단위로 다음과 같이 정교하게 섞습니다.
1. **Top 취향 존 (60%):** 유저 선호도 1~5위 장르에서 가장 인기 있는 영상 12개 추출.
2. **중간 발견 존 (20%):** 선호도 10~15위 장르에서 4개를 추출하여 **취향 확장(Discovery)** 유도.
3. **트렌드 존 (20%):** 취향과 무관하게 지금 가장 핫한 대중적 화제작 4개 배치.
- **라운드-로빈(Round-Robin) 수집:** 특정 한 장르만 연속으로 나오지 않도록 카테고리별로 1개씩 교차 수집하여 다양성을 확보했습니다.

---

## 3. Redis 기반 선제적 충전 시스템 (Proactive Buffer)

### 💡 기획 의도: "끊김 없는(Seamless) 무한 스크롤 구현"
무거운 추천 연산을 매번 수행하면 로딩 지연이 발생합니다. 이를 위해 **Redis 인메모리 캐시**를 활용한 선제적 대응 시스템을 구축했습니다.

### ⚙️ 기술적 로직 (Look-ahead Recharging)
- **인비저블 큐(Invisible Queue):** 유저별로 개인화된 영화 ID 리스트를 Redis에 저장합니다.
- **선제적 충전:** 유저가 10개를 소비하여 리스트 끝이 가까워지면, 서버는 이를 감지하고 **백그라운드에서 즉시 다음 20개를 믹싱하여 큐에 추가(`rpush`)**합니다.
- **데이터 정제 (3초 룰):** 유저가 3초 미만으로 스킵한 영상은 무시하고, 3초 이상 본 영상만 취향 데이터로 누적하여 추천의 순도를 높였습니다.

---

## 💎 기술적 요약 (발표 마무리용)
- **성능:** Redis를 활용하여 복잡한 추천 연산을 API 응답 시간 밖으로 분리(비동기적 효과).
- **정교함:** 단순 장르 매칭이 아닌 혼합 장르 평균 점수 산정 로직 적용.
- **사용자 경험:** 3초 룰과 하이브리드 믹스 전략을 통한 높은 추천 신뢰도 확보.
